# [`style` Prop](https://reactjs.org/docs/dom-elements.html#style)

```jsx
<div
  style={{
    padding: '1em',
    background: 'pink',
    color: 'blue'
  }}
>
  I am styled!
</div>
```

</Layout>

---

<Layout>

## Why Use the `style` Prop

<ul>
  <Appear>
    <li>Itâ€™s right there</li>
    <li>No extra steps needed</li>
    <li>Styles right there with the component</li>
    <li>Full confidence when deleting old styles</li>
    <li>Just using Javascript+JSON objects</li>
    <li>Inheritance gets you a long way</li>
    <li>Dynamic styling</li>
    <li>Theming is possible</li>
  </Appear>
</ul>

</Layout>

---

<Layout>

## Why <em>not</em> to use the `style` Prop

<ul>
  <Appear>
    <li>No media queries (@media)</li>
    <li>No CSS animations (@keyframes)</li>
    <li>No pseudo classes (:hover, :first)</li>
    <li>No pseudo elements (:before, :after)</li>
    <li>No web fonts (@font)</li>
    <li>No autoprefixer</li>
    <li>Additional costs in JS payload</li>
    <li>Debugging can be difficult</li>
    <li>No cascade</li>
  </Appear>
</ul>

</Layout>

## Benefits of CSS:

<ul>
  <Appear>
    <li><a href="https://wattenberger.com/blog/css-cascade" target="_blank" rel="nofollow">The Cascade!</a></li>
    <li>Fully-featured with native browser support</li>
    <li>High probability of developers being familiar with CSS</li>

    <li>You may already have existing styles to work with</li>

<Layout>

# Downsides of vanilla CSS

<ul>
  <Appear>
    <li>The Cascade!</li>
    <li>CSS can be hard to edit or remove</li>
    <li>No dynamic styling</li>
    <li>Often CSS is sent to the user as one large file</li>
    <li>Lots of tooling needed to create production-ready code</li>
    <li>Theming is complicated</li>
  </Appear>
</ul>

</Layout>
